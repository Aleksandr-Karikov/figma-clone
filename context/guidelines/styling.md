# Styling Guidelines

Tailwind CSS, CSS modules, and design system.

---

## Overview

**Styling Stack:**

- Tailwind CSS for utility classes
- CSS Modules for complex components
- CSS Variables for theming

---

## Tailwind CSS

### Basic Usage

**Pattern:**

- Use utility classes
- Combine classes for styling
- Follow design system tokens

### Conditional Classes

**Pattern:**

- Use `cn()` utility for conditional classes
- Combine Tailwind classes
- Handle variants

---

## CSS Modules

### For Complex Components

**Pattern:**

- Use for complex styling
- Scoped styles
- Import as module

---

## Design Tokens

### Colors

**Pattern:**

- Use design system colors
- Consistent color palette
- Support dark mode

### Spacing

**Pattern:**

- Use standard spacing scale
- Consistent spacing values

---

## Theming

### Dark Mode

**Pattern:**

- Support dark mode
- Use Tailwind dark mode classes
- CSS variables for theming

---

## Responsive Design

**Pattern:**

- Use Tailwind responsive breakpoints
- Mobile-first approach
- Test on different screen sizes

---

## Component Styling Patterns

### Button Component

**Pattern:**

- Variant system
- Size system
- State styles (hover, disabled)

---

## Animation

**Pattern:**

- Use Tailwind transitions
- CSS for complex animations
- Smooth user experience

---

## Best Practices

### ✅ Do

- Use Tailwind for most styles
- Use CSS Modules for complex components
- Use CSS Variables for theming
- Follow design system

### ❌ Don't

- Don't use inline styles (except dynamic values)
- Don't create custom classes unnecessarily
- Don't mix Tailwind and CSS Modules without reason
